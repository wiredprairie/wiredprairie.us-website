Windows Azure<\/a>. Admittedly, quite cool. I don\u2019t have any uses for it right now, but I like the fact that it\u2019s fully .NET friendly. The bad: a confusing mix of technologies somehow \u201clumped\u201d under this new brand-name. When are they ever going to learn?<\/p>\n

Windows 7 (engineering blog here<\/a>). The good: a refreshed UI and a bunch of things that hopefully will make it faster (like Vista should have been). The bad: showing a cool demo of the new UI at the PDC 2008 main session, yet giving attendees an older build that contained the least interesting pieces. Seriously bad form Microsoft.<\/p>\n

XNA Studio 3.0<\/a> released (coincidence actually), cool nonetheless. Makes it so much easier to write games than ever before for the Windows, XBox360, and Zune. <\/p>\n

Universal Studios Event. I\u2019d been there before several times, so there really wasn\u2019t much to do, and I\u2019m totally not into \u201chaunted\u201d mazes and such. The bad: there was a lot less fun food options than there had been in the past. Even worse: at least one bus got lost on the way from the hotels to the park (and it took about 1.25 hours to arrive!).<\/p>\n

Visual Studio 10. The UI will of course benefit from the switch to WPF \u2013 including some slick ways of enhancing the editor. Maybe some nice open source will appear which will all but eliminate the need to buy expensive 3rd party options for editor and shell enhancements (sorry, but they\u2019ve always been too pricey for me).<\/p>\n

Microsoft Surface: Yawn. Multi-touch in Windows 7 is where it\u2019s at. Plus, you\u2019re not hindered by the horizontal-only user experience and expensive tables.<\/p>\n
