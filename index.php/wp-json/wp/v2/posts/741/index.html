IE8 includes a new syntax highlighting View Source tool similar to what is available in other web browsers. However, you may not want to use the built-in, view-only tool. It\u2019s easy to change.<\/p>\n

1)<\/p>\n

Start Internet Explorer 8 if it isn\u2019t already started. Within IE8, click the Tools <\/strong>menu, then select Developer Tools <\/strong>(or just press [F12]).<\/p>\n

<\/p>\n

2) Then, from the developer tools, select File<\/strong>, Customize Internet Explorer View Source<\/strong>, and then Other\u2026<\/strong>.<\/p>\n

<\/p>\n

A file open dialog is displayed. Navigate to the application you want to use \u2026 for example Visual Studio 2008:<\/p>\n

<\/p>\n

That\u2019s it!<\/p>\n

 <\/p>\n

Note (how to completely reset Internet Explorer 8 View Source option)<\/strong>:<\/p>\n

When I first installed IE8, after having installed earlier builds of IE8 (beta and RC), I wasn\u2019t able to reset the view source option back to the Default Viewer. (I\u2019ve later noticed that even after selecting Default Viewer, it never actually leaves the Default Viewer option selected, even if that is the current option). After some registry hunting, you may need to clear out a registry setting that is incorrectly set.<\/p>\n

HKEY_CURRENT_USER\\Software\\Classes\\VirtualStore\\MACHINE\\SOFTWARE\\Microsoft\\Internet Explorer\\View Source Editor\\Editor Name<\/p>\n

<\/p>\n

The Key\u2019s (Default) value was set to Notepad on my machine, which would cause IE8 to always use Notepad to view source, even though I wanted to use the built-in view source option. I cleared the value (but did not remove the key) and everything began working as normal again. Using the menu option within the developer tool\u2019s in IE8 does not appear to set this value correctly, even though it is used as a potential supplying key\/value.<\/p>\n
