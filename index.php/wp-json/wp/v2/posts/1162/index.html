There was a huge uproar in the tech blogs this past week regarding some comments made by Bob Muglia (of Microsoft) as reported by Mary Joe Foley in a post entitled, \u201cMicrosoft: Our Strategy with Silverlight has shifted<\/a>.\u201d <\/p>\n

The quotes that caused the most ruckus are:<\/p>\n

\n

\u201cSilverlight is our development platform for Windows Phone<\/a>,\u201d he said. Silverlight also has some \u201csweet spots\u201d in media and line-of-business applications, he said.<\/p>\n

 <\/p>\n

But when it comes to touting Silverlight as Microsoft\u2019s vehicle for delivering a cross-platform runtime, \u201cour strategy has shifted,\u201d Muglia told me.<\/p>\n

 <\/p>\n

Silverlight will continue to be a cross-platform solution, working on a variety of operating system\/browser platforms, going forward, he said. \u201cBut HTML is the only true cross platform solution for everything, including (Apple\u2019s) iOS platform,\u201d Muglia said.<\/p>\n<\/blockquote>\n

OK, so many who read this read a lot between the lines. <\/p>\n

    \n
  • Microsoft Has Seen The Light. And It\u2019s Not Silverlight.<\/a> <\/li>\n<\/ul>\n

    Whatever HTML5 means to you (and your definition is likely different than mine unfortunately), it can\u2019t do everything. There\u2019s still a place for Silverlight and Flash in this world. Many consumer facing web sites, simply don\u2019t need Flash or Silverlight. Either of these browser plug-ins can enhance the consumer experience, but rarely are they needed for the average web site. You won\u2019t likely see more Flash or Silverlight on traditional consumer facing web sites.<\/p>\n

    Thankfully, BobMu responded<\/a> and tried to calm some fears (which was only somewhat effective).<\/p>\n

    I\u2019d consider there are a few places where HTML (4+) is not yet rich enough to provide the type of experience demanded by users:<\/p>\n

      \n
    • Games<\/strong>. JavaScript and HTML just isn\u2019t yet up to the task of creating a good rich game. It\u2019s not really the platform\u2019s fault \u2013 it just wasn\u2019t designed with it in mind. <\/li>\n
    • Business Applications. <\/strong>It\u2019s great that Twitter can run completely in the web browser or that you\u2019re able to enter a street address to collect a consumer\u2019s home shipping address for a package (with the ability to provide some text hinting as a bonus). But, business applications are far more complex, and require robust data and input handling that are very challenging to handle, especially cross browser. Layout of complex data (tables or not) can be maddening and extremely inefficient in browsers, when data needs to be truncated, formatted, centered, sorted, etc.\n

      It\u2019s easy to build a ineffective web application in HTML that doesn\u2019t consider the user. Too easy. That\u2019s where the plug-ins can excel when used properly.<\/li>\n<\/ul>\n

      Many things can be hacked to work in the web browser without Silverlight. However, this comes at a great cost of development and maintenance and QA. How many times have you written a web application and not needed to fix an issue that happens only in one browser (and worse, only in a particular version)? If you haven\u2019t, it\u2019s likely you haven\u2019t written a complex web application or that your needs are very simple. <\/p>\n

      This is where Silverlight (and Flash\/Flex) both shine. They provide a consistent and robust user experience across supported browsers and platforms. Write it once, and it really should run nearly anywhere the plug-ins are supported without issue. Furthermore, they both provide a better debugging and development experience than JavaScript\/HTML. <\/p>\n

      Even though Silverlight and Flex are best used in Enterprise development, I worry that Microsoft may still abandon the platform sooner rather than later.<\/p>\n

      Microsoft totally missed the boat on Silverlight for games. If they had gotten a better foothold in that market, I\u2019d be much less worried. I\u2019ve never encountered a game that was written in Silverlight that wasn\u2019t pointed by some Silverlight blog I follow. Flash I encounter all the time. Silverlight should be a decent gaming platform, but the development and design tools for gaming in Silverlight suck<\/strong> <\/strong>in comparison to those provided by Flash. Suck<\/strong>. There\u2019s no real consideration for the type of animation and character creation that the Flash tools provide that game development shops need.<\/p>\n

      Blend \u2013 phhpt. Don\u2019t get me started on how awful that would be to use as an animator. Expression Design? Ha!<\/strong> That thing is a joke compared to Flash CS5. At best, an animator could painfully create illustrations in Photoshop or Illustrator and then import them as layers into Blend, and then try to manage them \u2026 yuck. Too many tools and not enough interaction between them. <\/p>\n

      This is a new game that demonstrated at Adobe Max \u2013 beautifully done. It\u2019s using Flash. I can\u2019t imagine it being done in any other platform today.<\/p>\n
