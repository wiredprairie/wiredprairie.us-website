So that I don\u2019t forget how to tweak PostBox<\/a> again, I\u2019m posting a few custom configuration settings not available in Tools<\/strong>\/Options<\/strong> directly that I find useful. <\/p>\n

If you don\u2019t want to see the somewhat annoying status and progress dialog in Postbox (or Mozilla Thunderbird) anymore, there\u2019s a change you can make to hide the status window (dialog) entirely.<\/p>\n

Sending Messages, Status, Progress<\/p>\n

<\/p>\n

Open the Advanced Configuration Config Editor <\/strong>option found in Tools<\/strong>\/Options<\/strong>\/Advanced<\/strong>\/General:<\/strong><\/p>\n

<\/p>\n

When you open the Configuration Editor, you\u2019ll be shown a warning by default:<\/p>\n

<\/p>\n

As it suggests, be careful when you make these low level settings changes. Preferences that have changed from the default are shown in bold <\/strong>(preferences could have been changed by a variety of mechanisms, so don\u2019t panic when you see lots of non-default preferences).<\/p>\n

Here\u2019s how to hide the Sending Messages<\/strong> status window and instead show the status at the bottom of the message being sent (in the status bar). Using the configuration editor, change mailnews.show_send_progress<\/strong> to false<\/strong>. To quickly find the setting, type \u201cshow_send_p\u201d in the filter:<\/p>\n

<\/p>\n

Then, double-click on the row to toggle the Value from the default true<\/strong> to false.<\/strong> <\/p>\n

If you want Postbox\u2019s (or Thunderbird\u2019s) delete message behavior to more closely match Microsoft Outlook\u2019s, you may want to make this other change. Using the same technique as above to make a custom configuration edit, set the following preference to false.<\/strong><\/p>\n

mail.delete_matches_sort_order<\/strong><\/p>\n

When this is set, pressing the Delete key will not move to the next message (down) in the list. It will stay in place. <\/p>\n
