Michael suggests\u2026<\/p>\n

\n

It’s easy to look at the world today and say that web applications have won. This is web developer arrogance. Stupidity is to think that web applications have won because web applications are superior to desktop applications. Smarter, but probably still arrogant developers would point to web applications as disruptive technologies<\/a>. This involves admitting that web applications are inferior, but good enough, and present enough other "cheaper" advantages to compensate for their inferiority.<\/p>\n

To understand why the "web apps have won" claim is dubious. There are definitely a lot of awesome web applications out there. Many of them were created back in the mid\/late 90’s, The "features" of these applications were the key to applications, not the user interface. Now these days, most of these web applications offer APIs\/web services\/RESTful interfaces\/whatever you want to call them. In many cases it is possible to build desktop applications that tap into the same features as these web applications. However, this was certainly not the case 10-15 years ago.<\/p>\n<\/blockquote>\n

If a vendor could make installing and updating applications as seamless as web application models, I believe they\u2019d have a goldmine on their hands. <\/p>\n

Ease of deployment is what makes web applications rock<\/strong>. It\u2019s not their functionality, or rounded buttons, or big text boxes, or gradients. <\/p>\n

You don\u2019t need to write a web application like the often praised 37 Signals\u2019 applications (such as Basecamp<\/a>) to have a well received application. It\u2019s not because it\u2019s a web application. It\u2019s because they put in the features and functionality that customers need. If they\u2019d written the same in a desktop application, and it was as simple to deploy and use, I\u2019d argue they\u2019d be just as successful.<\/p>\n","protected":false},"excerpt":{"rendered":"
