If you didn\u2019t see the buzz about the Internet in the last 24 hours \u2013 where have you been? :) <\/p>\n

Google announced officially<\/a> that they are working on a new operating system, named Google Chrome OS <\/strong>(just rolls off the tongue doesn\u2019t it?).<\/p>\n

The few interesting points made in the announcement blog posting:<\/p>\n

    \n
  1. Targeted initially at netbooks<\/li>\n
  2. Open source, Linux based with a new windowing system and not Android based as this will work on ARM and x86 chips.<\/li>\n
  3. Fast and lightweight \u2013 only a browser \u2013 all running within the Chrome web browser on modern standards<\/li>\n
  4. No worries about drivers and software updates<\/li>\n
  5. Released available fall 2009 initially<\/li>\n<\/ol>\n

    The point that interests me is #4, as the rest of it isn\u2019t a surprise at all.<\/p>\n

    I\u2019m not sure I understand how they\u2019ll handle the various cameras, printers, mice, etc. that everyone will still want to use? That\u2019s where Windows has a huge advantage today \u2013 the wide availability of hardware for Windows. If users can\u2019t plug in their camera and just have it work, they\u2019ll be frustrated (and print it of course). Thankfully, there are more standards around camera connections these days \u2013 but \u2026.<\/p>\n

    What about RAW file formats for those of that don\u2019t shoot JPEG for example? There are frequent updates from Adobe\/Nikon\/Canon\/etc. to handle the constant stream of new file formats and changes that occur from each manufacturer. Maybe they\u2019ve got this all figured out \u2013 but my experience with Linux in the past is that far too often a piece of hardware that worked fine with Windows isn\u2019t recognized by the Linux. <\/p>\n

    I wonder what Microsoft will do about Silverlight? I\u2019d be disappointed if it didn\u2019t work on this new class of devices if it takes off. But, I also wonder about Flash support. Just because it\u2019s Linux doesn\u2019t mean that Flash will work. Google believes in HTML 5 so much that I can see them not being very \u201copen\u201d about making it work.<\/p>\n","protected":false},"excerpt":{"rendered":"
