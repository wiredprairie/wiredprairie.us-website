I\u2019ve been asked by a few people for more details on the API Nest Labs uses for their thermostats, especially regarding setting data (and not just polling).<\/p>\n

The API uses mostly JSON formatted data POSTed to their web servers.<\/p>\n

Authentication<\/h2>\n

To authenticate, POST the username and password, encoded as form url-encoded:<\/p>\n

\n
POST https:\/\/home.nest.com\/user\/login HTTP\/1.1<\/span>\nHost: home.nest.com\nProxy-Connection: keep-alive\nAccept-Encoding: gzip, deflate\nContent-Type: application\/x-www-form-urlencoded; charset=utf-8\nAccept-Language: en-us\nContent-Length: {Length}\nAccept: *\/*\nConnection: keep-alive\nUser-Agent: Nest\/3.0.1.15 (iOS) os=6.0 platform=iPad3,1\n\nusername={email}&password={password}<\/pre>\n<\/div>\n
Adjust the email and password, and the content length to fit. You may need to remove the Accept-Encoding header value if your client cannot accept gzip or deflated responses.<\/p>\n
The server responds with a healthy set of basic information (in JSON format):<\/p>\n
\n
{\n    "is_superuser"<\/span>: false<\/span>,\n    "is_staff"<\/span>: false<\/span>,\n    "urls"<\/span>: {\n        "transport_url"<\/span>: "https:\/\/{subdomain}.transport.nest.com:443"<\/span>,\n        "rubyapi_url"<\/span>: "https:\/\/home.nest.com\/"<\/span>,\n        "weather_url"<\/span>: "http:\/\/www.wunderground.com\/auto\/nestlabs\/geo\/current\/i?query="<\/span>,\n        "support_url"<\/span>: "https:\/\/nest.secure.force.com\/support\/webapp?"<\/span>\n    },\n    "limits"<\/span>: {\n        "thermostats_per_structure"<\/span>: 10,\n        "structures"<\/span>: 2,\n        "thermostats"<\/span>: 10\n    },\n    "access_token"<\/span>: "GIANT TOKEN STRING=="<\/span>,\n    "userid"<\/span>: "1234"<\/span>,\n    "expires_in"<\/span>: "Wed, 07-Oct-2012 12:08:00 GMT"<\/span>,\n    "email"<\/span>: "user@example.com"<\/span>,\n    "user"<\/span>: "user.1234"<\/span>\n}<\/pre>\n<\/div>\n
There are a few things you\u2019ll need from the response:<\/p>\n
    \n
  • transport_url<\/strong> : this is the address for all of the later request that are made. I\u2019d speculate it\u2019s just a specific server in a server farm (likely with server affinity\/session) <\/li>\n
  • access_token<\/strong> : this is the key for all later requests and grants access to the API <\/li>\n
  • userid\/user <\/strong>: a unique user ID <\/li>\n
  • expires_in<\/strong> : this is the timestamp for when the access token expires <\/li>\n<\/ul>\n
    I\u2019m not sure why the \u201climits\u201d are being sent back to the client.<\/p>\n
    You can obtain the service URLs at any time:<\/p>\n
    \n
    POST https:\/\/home.nest.com\/user\/service_urls HTTP\/1.1<\/span>\nHost: home.nest.com\nAuthorization: Basic GIANT TOKEN STRING==\nAccept-Encoding: gzip, deflate\nAccept: *\/*\nContent-Length: 0\nAccept-Language: en-us\nConnection: keep-alive\nProxy-Connection: keep-alive\nUser-Agent: Nest\/3.0.1.15 (iOS) os=6.0 platform=iPad3,1<\/pre>\n<\/div>\n
    Just insert an Authorization header with the access_token<\/strong> value.<\/p>\n
    The response:<\/p>\n
    { <\/span>"urls"<\/span>: { <\/span>"transport_url"<\/span>: <\/span>"https:\/\/{subdomain}.transport.nest.com:443"<\/span>, <\/span>"rubyapi_url"<\/span>: <\/span>"https:\/\/home.nest.com\/"<\/span>, <\/span>"weather_url"<\/span>: <\/span>"http:\/\/www.wunderground.com\/auto\/nestlabs\/geo\/current\/i?query="<\/span>, <\/span>"support_url"<\/span>: <\/span>"https:\/\/nest.secure.force.com\/support\/webapp?" <\/span>}, <\/span>"limits"<\/span>: { <\/span>"thermostats_per_structure"<\/span>: 10, <\/span>"structures"<\/span>: 2, <\/span>"thermostats"<\/span>: 10 } }<\/span><\/pre>\n
    Nest labs has a special URL at wunderground.com to access the weather.<\/p>\n
    One of the first requests you might want to send is to get a complete<\/strong> picture of the system:<\/p>\n
    \n
    GET https:\/\/{subdomain}.transport.nest.com:443\/v2\/mobile\/#USER.ID# HTTP\/1.1<\/span>\nHost: {subdomain}.transport.nest.com:443\nAuthorization: Basic GIANT TOKEN STRING==\nAccept: *\/*\nAccept-Encoding: gzip, deflate\nAccept-Language: en-us\nConnection: keep-alive\nX-nl-protocol-version: 1\nX-nl-user-id: #USERID#\nProxy-Connection: keep-alive\nUser-Agent: Nest\/3.0.1.15 (iOS) os=6.0 platform=iPad3,1<\/pre>\n<\/div>\n
    You\u2019ll make the request to the transport_url<\/strong> and make sure that the Host, Authorization, and X-nl-user-id header values are set appropriately. The Url now must include though:<\/p>\n
      \n
    • version <\/li>\n
    • mobile <\/li>\n
    • and the full user id (like user.1234) <\/li>\n<\/ul>\n
      So, it will look something like: \/v2\/mobile\/user.1234<\/strong><\/p>\n
