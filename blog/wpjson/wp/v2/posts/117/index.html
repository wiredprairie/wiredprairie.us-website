Window Handles that is. <\/p>\n

WPF Window Handles.<\/p>\n<\/blockquote>\n

It’s not hard to get the HWND for a WPF Window. There’s a class that makes it easy to get access to the Win32 based HWND that hosts a WPF Window, called WindowInteropHelper.<\/p>\n

I saw this<\/a> post by Brandon and it made me think, Extension Methods to the rescue!<\/p>\n

His solution, although neat, means that there’s an extra object that needs to be created just so that a properties value could be delay loaded. Window handles generally should be obtained and discarded anyway (as they can change), so I thought I’d go with this method to add the Handle property to every WPF Window, without needing to add either a custom class and derive everything from that, or manually add it to every class.<\/p>\n

public static class <\/span>WindowExtension\n<\/span>{\n    public static <\/span>IntPtr <\/span>GetHandle(this <\/span>Window <\/span>w)\n    {\n        return new <\/span>WindowInteropHelper<\/span>(w).Handle;\n    }\n}<\/pre>\n
<\/a>To use:<\/p>\n
this<\/span>.GetHandle()<\/pre>\n
<\/a>Done. I don’t mind it being a method in this case as it’s not necessarily a fast operation, and it’s read-only. <\/p>\n
