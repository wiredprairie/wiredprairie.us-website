Apple makes user experience inroads every day on Windows when a developer follows Apple\u2019s Human Interface Guidelines for design for a Windows Application. <\/p>\n

This post was inspired when I installed Miro<\/a> this morning on my desktop PC. (Don\u2019t get me started that it opts-in <\/strong>a bunch of changes and a toolbar for the browser. Convince me that I want to to do it because it\u2019s the \u201cright\u201d thing \u2013 let me opt-in!)<\/p>\n

In Miro\u2019s words, it\u2019s an \u201camazing open-source, non-profit video player.\u201d (See previous annoyance about toolbars and other default-opt-in things the installer attempts).<\/p>\n

In any case, it installed. The installer UI had a few cross platform oddities, including not being sized correctly, and worse, showing up too high on one of my monitors. <\/p>\n

On the left side (in both applications), is located a Source List (see \u201cSource Lists<\/a>\u201d in the Apple Human Interface Guidelines). They flipped the play bar to the bottom and tweaked the general look and feel of the buttons, but it\u2019s basically the same UI. <\/p>\n

At first, you might think that there\u2019s only so many ways to present that type of UI. I give you the Zune Player (4.7).<\/p>\n

<\/p>\n

It\u2019s so different I won\u2019t even start listing the differences. I thought about other applications that I have on my Windows PC that follow this style. Admittedly, it\u2019s not very many right now. MetroTwit generally follows the Metro style. <\/p>\n

Actually, there\u2019s not a formal guide for this style. The closet you might come today is the Windows Phone 7 UI Design and Interaction Guide<\/a>. That\u2019s a poor substitute though.<\/p>\n

There is of course, the Windows User Experience Interaction Guidelines<\/a>. Oh wow. It\u2019s amazing the amount of bad examples that are sourced from earlier versions of Windows. There are so many (too many). Furthermore, it\u2019s not free from errors itself:<\/p>\n

<\/p>\n

Microsoft doesn\u2019t bother following it, so it\u2019s not too surprising that many Windows applications do not. <\/p>\n

From Word 2010:<\/p>\n

<\/p>\n

My point isn\u2019t to blast this document. What\u2019s missing from the document is how to build an application that is smooth and modern. It\u2019s an 882 page document currently, that helps you build a nice Windows 95 application. What? Seriously. If you look through the document, it\u2019s really about fixing the problems of Windows 95 through Windows Vista. It doesn\u2019t innovate anything new. It\u2019s about putting up better error messages, being more consistent, etc. All of these things are great. But, can we move beyond that? Can we build something slick and modern that is more than just gas<\/strike> glass? <\/p>\n

That\u2019s why I\u2019m pushing for anything more modern now. If the Zune\/Metro isn\u2019t it, then create something new for us to follow. <\/p>\n

Applications for developers\/designers that are released like WebMatrix are a sign that there\u2019s an opportunity to think beyond just glass and better wording. In WebMatrix, I noticed that they\u2019ve done away with standard Modal dialogs. They\u2019re modal \u2013 but they float with the main application window (even when they\u2019re showing), much like if they were a web page. Slick. The application is clean and modern. I like it.<\/p>\n

<\/p>\n

instead of this:<\/p>\n

<\/p>\n

So, this was just a rant really. But, an important one. Will Windows 8 finally solve this and introduce a awesome and modern User Experience, one that is easily replicated by developers, without the use of dozens of third party components and questionable code from the Internet, and \u2026. Please Microsoft. Do it. Do it for all of us<\/strong>. I don\u2019t care if it\u2019s the Metro\/Zune theme or something else. You can do better than Apple. <\/p>\n

It\u2019s time for you to innovate again. You\u2019ve got some super app creation platforms (like Silverlight and WPF) and you\u2019ve got awesome development tools (Visual Studio and .NET 4), and you\u2019ve got a lot of great developers ready to rock and roll. Help us build UX that are killer awesome. Build the components \u2013 ship them in V1. Do it!<\/p>\n
